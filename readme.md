## 项目概述

这是一个基于DeepSeek API的智能对话系统，具有多语言支持、记忆功能、命令执行和自动交互等特色功能。系统采用模块化设计，包含核心逻辑处理、AI交互、音频处理和GUI界面等组件。

## 特色功能

### 1. 多语言支持

- 支持中日双语显示（中文和日文）
- 可配置是否显示双语内容
- 日语文本自动生成语音输出

### 2. 智能记忆系统

- 记忆存储与检索功能
- 支持按时间范围查询记忆
- 记忆内容持久化存储
- 记忆会发生部分性遗忘，模仿人类记忆系统

### 3. 命令执行系统

AI回复以命令的形式出现，系统可以识别并执行命令。在命令执行过程中，系统可以根据配置的返回方式返回结果。

- 支持多种命令执行方式：
  - 立即返回结果(`immediately`)
  - 全部命令执行后返回(`afterall`)
  - 下一轮对话返回(`next_turn`)
  - 无需返回(`none`)
- 内置命令包括：
  - 与用户对话(`talk_to_user`)
  - 执行系统命令(`cmd_processor`)
  - 延时(`delay`)
  - 记忆管理(`add_memory`, `read_memories`)

### 4. 自动交互功能

- 可配置的自动消息间隔（默认15分钟）
- 智能判断系统忙碌状态
- 线程安全的自动消息处理

### 5. 可配置参数

- 温度参数(0-2范围)
- 最大上下文长度
- API密钥配置
- 系统提示词配置
- 是否使用本地/云端语音合成
- 记忆系统参数配置
- 自动消息间隔配置

## 实现方式

### 核心架构

1. **ChatCore** - 核心逻辑处理器
   - 管理消息队列和状态
   - 协调各组件工作
   - 处理命令执行流程

2. **AIHandler** - AI交互模块
   - 封装DeepSeek API调用
   - 处理历史记录管理
   - 解析AI响应为结构化命令

3. **MemoryHandler** - 记忆系统
   - 记忆的存储和检索
   - 时间范围查询支持
   - 模仿人类记忆系统的部分性遗忘

4. **GUI** - 用户界面
   - 基于Tkinter的图形界面
   - 状态显示和控制面板
   - 老旧的win风格界面

### 关键技术

- 多线程处理（消息、自动交互）
- JSON格式配置和历史存储
- 正则表达式解析AI响应
- 结构化命令处理流水线

## 拓展性

### 命令系统扩展

- 可轻松添加新的命令类型及其模块
- 支持多种返回处理方式
- 命令参数灵活可配置

## 潜在不足

### 1. 性能考虑

- 长时间运行可能导致历史记录过大
- 复杂命令处理可能阻塞主线程

### 2. 安全性

- 历史记录存储不加密
- 系统命令执行需谨慎处理
- API密钥明文存储
- 缺乏用户认证机制

### 3. 稳定性
- 网络异常处理不够完善
- 长时间自动运行可能产生内存泄漏
- 异常恢复机制有待加强

### 4. 功能局限

- 对话上下文管理较简单
- 记忆系统检索功能有限
- 缺乏对话状态跟踪，发生错误时难以确定原因

## 安装与使用

### 依赖安装
```
pip install -r requirements.txt
```

### 配置说明

1. 在`config.json`中配置API密钥和其他参数
2. 在`memory_config.json`中配置记忆系统参数
3. 首次运行会自动创建配置和历史文件
4. 语音合成目前使用的是LiteSovits，需要自行配置

### 运行方式

```
python main.py
```
